STOCK CONTROL

Allocate Stock
(Screen: STK002)

 


To access the Allocate Stock screen:
Select the Airline Functions tab. On the Stock Control menu, go to Stock and click Allocate.

 

The Allocate Stock screen can be used by Cargo Headquarters, Stock Control Manager, Regional Cargo Manager and Cargo Station Manager to allocate stock against the requests made by stock holders of a lower level

 

The From Date and To Date fields would be populated with the current date by default. The Status would be ALL

 

The screen provides links to Monitor Stock and Allocate New Stock screens

Approve a stock request

1.        Select the Airline Functions tab. On the Stock Control menu, go to Stock and click Allocate to get the Allocate Stock screen.

 

2.        Enter the details of the requests to be listed. The following fields are mandatory:

 

-           Doc Type

-           Sub Type

-           Stock Control For

-           Status

-           From Date

-           To Date

 

3.        Click List.

The matching stock requests for the criteria specified are displayed in the table.

 

4.        Edit the Appr Stock field of the required request to enter the number of stocks to be approved.

Note:
The specified number of stocks for approval can be greater than the requested number of stocks.

 

5.        Select the stock request and click Approve.

The specified number of stocks is approved and the status of the request changes to Approved.

 

Field/Button Description - Allocate Stock screen

 

Allocate stock to a stock holder against a stock holder request

 

1.        Select the Airline Functions tab. On the Stock Control menu, go to Stock and click Allocate to get the Allocate Stock screen.

 

2.        Enter the details of the requests to be listed. The following fields are mandatory:

 

-           Doc Type

-           Sub Type

-           Stock Control For

-           Status

-           From Date

-           To Date

 

3.        Click List.

The matching stock requests for the criteria specified are displayed in the table.

 

4.        Edit the Appr Stock field of the required request to enter the number of stocks to be approved.

Note:
The specified number of stocks for approval can be greater than the requested number of stocks.

 

5.        Select the stock request and click Approve.

The specified number of stocks is approved and the status of the request changes to Approved.

 

6.        Select the approved request.

 

7.        Click List next to the Start Range field .

Available stock ranges with the approver, starting from the start range specified  are displayed in the table below. The total number of documents is equal to the To Be Alloc stock, if that many stocks are available with the approver. If the number of documents that are available with the approver is less than what is requested, then whatever is available is listed.

Note:
If the user wants the system to display the stock range starting from a particular number, then specify the start range in the right side Table section and click List.

 

8.        Edit the range specified, if required, before allocation, by clicking Edit Range .

Field/Button Description - Edit Range screen

 

9.        Click Allocate.

The displayed stocks are allocated to the requested stockholder.

Note:
-     If the actor allocates all the approved stock, then the system updates the status as Allocated.
-     If only a part of the approved stock is allocated, then the system maintains the status as Approved.
-     If the allocated stock, approved stock and requested stock are same (approved stock can be greater than requested stock), system updates the status as ‘ Complete’. Then it is removed from the displayed list of stock requests and can be viewed in the List Stock Requests screen

 

Field/Button Description - Allocate Stock screen

 

Edit the stock range to be allocated against a stock holder request

 

1.        Select the Airline Functions tab. On the Stock Control menu, go to Stock and click Allocate to get the Allocate Stock screen.

-     In the Allocate Stock screen, specify the Doc Type, Sub Type, Stock Control For, Status, From Date and To Date and click List.

-     Select the request for which allocation is to be done and click List in the right side Table section.

-     Click Edit Range to get the Edit Range screen

 

2.        The Edit Range screen appears with the details of the stock ranges that are to be allocated, which is displayed in the Allocate Stock screen.

Note:
If some other range than the ones displayed in the screen is to be displayed, specify the Range From and Range To details
                                          OR
  Range From and No of Docs details

 

3.        Click List.

The range of documents available within the specified range is displayed.

The total number of documents available for allocation is displayed in the Total no of Docs field.

 

4.        Select the range to be edited and click >> to move it to the right side Table section.

 

5.        In this section, edit the Range From and Range To values to the desired stock range to be allocated.

Note:
-     In this Table section, more rows can be added if required by clicking Add, to specify different ranges of stocks.
-     The total number of stocks that has been allocated is displayed in the Total no of Docs field.

Field/Button Description – Edit Range screen

 

6.        Click OK.

The newly edited range is displayed in the right side table section in the Allocate Stock screen.

Complete a stock request

 

During the allocation of stocks, if the number of allocated stock, approved stock and requested stock are same (approved stock can be greater than requested stock), system itself updates the status of the corresponding request as ‘ Completed’ . Apart from this, the status change can be done manually also, if no further action is intended to be taken on a request

 

1.        Select the Airline Functions tab. On the Stock Control menu, go to Stock and click Allocate to get the Allocate Stock screen.

 

2.        Enter the details of the requests to be listed. The following fields are mandatory:

 

-           Doc Type

-           Sub Type

-           Stock Control For

-           Status

-           From Date

-           To Date

 

3.        Click List.

The matching stock requests for the criteria specified are displayed in the table.

 

4.        Select the required stock request and click Complete.

 

5.        Click Yes in the confirmation message that appears.

The request status is changed to Completed and it is removed from the displayed list of stock requests.

Note:
-     Requests can be completed only if there is some allocated stock
-     To view the completed requests, go to the List Stock Requests screen

View list of stock requests in the List Stock Requests screen

Field/Button Description - List Stock Requests screen

 

6.        Click Close to exit the screen.

Description of Fields and Buttons

Optional - Fields in which data entry is optional

Display Field - Non-editable fields in which data appears automatically

SEARCH CRITERIA SECTION

 

Doc Type (Mandatory)

The type of the document whose details are to be searched and displayed

Select this from the drop-down list.

 

Sub Type (Mandatory)

The sub type of the document whose details are to be searched and displayed

Select this from the drop-down list.

Note:
The values available in this drop-down would depend upon the Doc Type selected. On selecting a Doc Type, the Sub Type field  gets automatically populated and this value can be changed  if required by selecting the required value from the drop down list.

 

Stock Control For (Mandatory)

The approver for the stock holder

The approver will be controlling the stock allocation.

Select this from the drop-down list.

 

Status (Mandatory)

Specify this to search for requests with the particular status.

The possible values are: New, Approved and Allocated. Select ALL to display the requests in any of these statuses

Select this from the drop-down list.

Note:
-     If the request is created and no action has been taken on that, the status will be New.
-     If only a part of the approved stock is allocated, then the system maintains the status as Approved.
-     If all the approved stock is allocated, then the system updates the status as Allocated.

 

From Date - To Date (Mandatory)

Specify this to search for requests made in this period

The current date would be displayed in these fields by default. This can be changed if required.

Specify these in the fields or select from the calendar.

 

StockHolder Type (Optional)

Specify this to search for requests made by a particular type of stockholder

Select this from the drop-down list.

 

StockHolder Code (Optional)

Specify this to search for requests made by a particular stockholder

Specify this in the field or select from the LOV.

Note:
The values displayed in this LOV depend upon the StockHolder Type selected. If no stockholder type is selected, then all the available values are shown.

 

Manual check box (Optional)

Select this check box  to search for requests made for printed stocks.

 

 

THE LEFT TABLE SECTION

 

Req Ref No (Display Field)

The system generated unique reference number, which identifies the displayed stock request

 

Req By (Display Field)

The stock holder who had raised the request

 

Date Of Request (Display Field)

The date on which the request was raised

 

Doc Type (Display Field)

The type of document that is requested for

 

Sub Type (Display Field)

The sub type of the document that is requested for

 

Req Stock (Display Field)

The number of stocks requested

 

Appr Stock (Optional)

The number of stocks approved among the stocks requested for

 

Alloc Stock (Display Field)

The number of stocks allocated among the stocks requested for

 

To Be Alloc (Display Field)

The number of stocks to be allocated among the stocks requested for

This is the difference between the approved stock and allocated stock.

 

Status (Display Field)

The status of the request

The possible values are: New, Approved and Allocated.

 

Manual (Display Field)

Indicates whether the request was made for manual stock

The possible values are Yes and No.

 

Approval/Rejection Remarks (Optional)

The remarks, if any, while approving / rejecting the stock request(s)

Specify this in the field.

 

THE RIGHT TABLE SECTION

 

Start Range (Optional)

The start range from which the available stocks are to be displayed

By default, zero is displayed in this field. This can be changed if required.

Specify this in the field.

 

Range From (Display Field)

The starting number of the document range available for allocation

 

Range To (Display Field)

The ending number of the document range available for allocation

 

No of Docs (Display Field)

The number of stocks in the range displayed, available for allocation

 

Remarks (Optional)

Remarks on the allocated stock, if any

Specify this in the field.
