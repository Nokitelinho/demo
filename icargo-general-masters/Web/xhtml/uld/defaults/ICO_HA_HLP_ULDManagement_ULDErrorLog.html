View list of ULD errors for UCM OUT/UCM IN messages

 

1.      Select the ULD Management tab. On the ULD Messages menu, click ULD Error logs to get the ULD Error Log screen.

 

2.      In the ULD Error Log screen, specify the search criteria. The following are mandatory:

·        Type the Flight No. (Flight Number)

·        Type or select the Flight Date.

·        Select the Message Type .

 

3.      Click List. (mandatory)

The list of ULD error details of the specified flight appear. The buttons and links become enabled.

 

4.      To exit the screen, click Close.

Reconcile ULD errors for UCM messages when ULD is not in stock

Errors can happen while trying to send a UCM OUT/UCM IN message for a ULD that is not in the stock. Such types of errors must be reconciled before trying to send the UCM message.

This type of error, ‘ULD is not in stock’ can occur because the ULD handling staff might have attempted to send a UCM message for a ULD which is presently not there in stock, in which case, the system will prompt the user to introduce the ULD to the particular origin station, by displaying the ‘ Maintain ULD’ screen.

 

1.      Select the ULD Management tab. On the ULD Messages menu, click ULD Error logs to get the ULD Error Log screen.

 

2.      In the ULD Error Log screen, list the ULD errors. (mandatory)

 

3.      In the table, select the error to be reconciled. (mandatory)

 

4.      Click Reconcile. (mandatory)

The Maintain ULD screen appears.

 

5.      In the Maintain ULD screen:

a)     Specify the necessary ULD details in order to introduce the ULD to the origin station.

Create a ULD

Field/Button Description – Maintain ULD screen

b)     Click Save.

     The ULD details are saved.

c)      Click Close.

The ULD Error Log screen reappears with the selected error reconciled.

 

6.      To exit the screen, click Close.

 Field/Button Description – ULD Error Log screen

 

Reconcile ULD errors for UCM messages when ULD is non operational

This type of error, ‘ULD is non operational’ can occur because the system must have kept the status of the ULD as non operational, when actually the ULD might have been put into operational use and when a UCM message is sent against the same, the discrepancy marked against the ULD is to be resolved in the Maintain Damage Report screen, where the overall status of the ULD is to be changed to ‘Operational’.

 

1.      Select the ULD Management tab. On the ULD Messages menu, click ULD Error logs to get the ULD Error Log screen.

 

2.      In the ULD Error Log screen, list the ULD errors. (mandatory)

 

3.      In the table, select the error to be reconciled. (mandatory)

 

4.      Click Reconcile. (mandatory)

The Maintain Damage Report screen appears with the details of the ULD.

 

5.      In the Maintain Damage Report screen:

a)     Select the Overall Status as ‘Operational’.

Field/Button Description – Maintain Damage Report screen

b)     Click Save.

The ULD details are saved.

c)      Click Close.

The ULD Error Log screen reappears with the selected error reconciled.

 

6.      To exit the screen, click Close.

 Field/Button Description – ULD Error Log screen

 

Reconcile ULD errors for UCM messages when ULD is not in station stock

This type of error, ‘ULD is not in station stock’ can occur because the ULD is not currently present in the station stock at that point of time but is present in another station when the UCM message was sent. This results in the ULD error, ‘ULD is not in station stock’ where the ULD will have to be moved from the current location to the station from where the UCM message was sent.

 

1.      Select the ULD Management tab. On the ULD Messages menu, click ULD Error logs to get the ULD Error Log screen.

 

2.      In the ULD Error Log screen, list the ULD errors. (mandatory)

 

3.      In the table, select the error to be reconciled. (mandatory)

 

4.      Click Reconcile. (mandatory)

The Record ULD Movement screen appears with the details of the ULD.

 

5.      In the Record ULD Movement screen:

a)     Specify the required details. The following are mandatory:

·        In the Flight Details table:

-          Type or selectthe Carrier Code

-          Type the No (Flight Number)

-          Type or selectthe Flight Date

-          Type or selectthe POL (Point Of Lading)

-          Type or selectthe POU (Point Of Unlading)

·        In the ULD Details table, type the ULD No (ULD Number)

·        Type or selectthe Current Airport

Field/Button Description – Record ULD Movement screen

Note:

The value of Current Airport should be changed to the airport from where the UCM message was sent.

 

b)     Select the Update Current Airport check box.

 

c)      Click Save.

The error is reconciled.

Note:

To discontinue the process and return to the ULD Error Log screen, click Close.

 

d)     Click Close.

The ULD Error Log screen reappears with the selected error reconciled.

 

6.      To exit the screen, click Close.

 Field/Button Description – ULD Error Log screen

 

Reconcile ULD errors for UCM messages when ULD is in transit

This type of error occurs when a ULD is assigned to a flight which is finalised. The same ULD can be assigned to another flight also, for which when a UCM OUT message is sent, a ULD error appears in the form, ‘ULD is in transit’, indicating that the ULD is in transit. This error can be resolved by adding the ULD to the flight for which the UCM OUT message is to be sent.

 

1.      Select the ULD Management tab. On the ULD Messages menu, click ULD Error logs to get the ULD Error Log screen.

 

2.      In the ULD Error Log screen, list the ULD errors. (mandatory)

 

3.      In the table, select the error to be reconciled. (mandatory)

 

4.      Click Reconcile. (mandatory)

Click Yes in the confirmation message that appears.

 

5.      The ULD Error Log screen refreshes with the selected error reconciled.

 

6.      To exit the screen, click Close.

 Field/Button Description – ULD Error Log screen

 

Delete ULD errors for a flight

 

1.      Select the ULD Management tab. On the ULD Messages menu, click ULD Error logs to get the ULD Error Log screen.

 

2.      In the ULD Error Log screen, list the ULD errors. (mandatory)

 

3.      In the table, select the required row to be deleted. (mandatory)

Note:

You can delete multiple ULD errors at a time. To do this, select all the corresponding rows.

 

4.      Click the Delete link. (mandatory)

 

5.      Click Save. (mandatory)

The selected ULD errors are deleted from the system.

 

6.      To exit the screen, click Close.
